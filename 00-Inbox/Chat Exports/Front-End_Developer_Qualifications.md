 Difference between a shallow copy and a deep copy when dealing with primitive and non-primitive data types in JavaScript?

### Assistant:
In JavaScript, the concepts of shallow copy and deep copy refer to different ways of creating copies of objects or arrays. The main difference between the two lies in how they handle nested data structures and whether they create a new independent copy or a reference to the original data.

1. Shallow Copy:
A shallow copy creates a new object or array but only copies the references to the nested data within the original object or array. In other words, it creates a new object, but the nested objects are still referenced from the original object. Shallow copying is the default behavior for most JavaScript operations.

When dealing with primitive data types (e.g., numbers, strings, booleans), a shallow copy creates a completely independent copy. Any modifications made to the copied primitive value will not affect the original value.

Example:
```javascript
const originalArray = [1, 2, 3];
const shallowCopy = originalArray.slice();

shallowCopy[0] = 10;
console.log(originalArray);  // Output: [1, 2, 3]
console.log(shallowCopy);    // Output: [10, 2, 3]
```

However, when dealing with non-primitive data types (e.g., objects, arrays), a shallow copy creates a new object or array, but the nested objects are still shared between the original and copied versions. Modifying a nested object in the copied version will reflect in the original object, as they share the same reference.

Example:
```javascript
const originalObj = { name: 'John', age: 30 };
const shallowCopy = Object.assign({}, originalObj);

shallowCopy.age = 40;
console.log(originalObj);    // Output: { name: 'John', age: 30 }
console.log(shallowCopy);    // Output: { name: 'John', age: 40 }

originalObj.name = 'Jane';
console.log(originalObj);    // Output: { name: 'Jane', age: 30 }
console.log(shallowCopy);    // Output: { name: 'Jane', age: 40 }
```

2. Deep Copy:
A deep copy, on the other hand, creates a completely independent copy of an object or array, including all nested objects or arrays. It recursively traverses the original data structure and creates new copies of all nested objects or arrays, ensuring that any modifications made to the copied data will not affect the original data.

To achieve a deep copy, you can use various methods such as recursion, `JSON.parse(JSON.stringify())`, or external libraries like Lodash's `cloneDeep`.

Example:
```javascript
const originalObj = { name: 'John', age: 30 };
const deepCopy = JSON.parse(JSON.stringify(originalObj));

deepCopy.age = 40;
console.log(originalObj);    // Output: { name: 'John', age: 30 }
console.log(deepCopy);       // Output: { name: 'John', age: 40 }

originalObj.name = 'Jane';
console.log(originalObj);    // Output: { name: 'Jane', age: 30 }
console.log(deepCopy);       // Output: { name: 'John', age: 40 }
```

It's important to note that deep copying can be more resource-intensive and may have limitations when copying certain data types like functions or symbols. Additionally, circular references within an object or array can cause issues with deep copying.
